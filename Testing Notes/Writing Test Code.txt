### Test Categories ###
EXPECTED BEHAVIOR - Does your code do what it should in friendly/general situations?

BOUNDARY/EDGE - Does adding the first item/last item/full collection break it?

FORBIDDEN INPUT - Does it work when people try to break your code w/ inputs?

EMPTY/NULL - Does it work if it's empty/null?  If its parameters are empty/null?

SCALE - Test with 1, 10, 100, 10000 inputs
